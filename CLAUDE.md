# QTC Kit Configurator ‚Äî Claude Context

## What This Is
A **Google Apps Script web app** for the Steelwrist sales team to look up whether a Quantum Connect (QTC) installation kit exists for a specific excavator brand and model. The backend is Google Sheets; the frontend is React + Vite bundled into a single HTML file served by GAS HtmlService.

**Live URL:** deployed as a Google Workspace domain-restricted web app (execute as user deploying, access: domain).

## Google Sheets Database

**Spreadsheet ID** stored in GAS Script Properties (`SPREADSHEET_ID`). 5 sheets:

| Sheet | Columns |
|-------|---------|
| Brands | id, name, logoFilename, active, createdAt |
| Models | id, brandId, brandName, name, tonnage, machineType, createdAt |
| Kits | id, modelId, status, cableLength, leftJoysticks, rightJoysticks, needsFeederValves, needsExtraQio, steeringKits, configPartNumber, prerequisites, limitations, updatedAt, updatedBy, **cableKitPartNumber**, **cableKitDescription** |
| KitRequests | id, brandName, modelName, requestedBy, note, status, createdAt, adminNote |
| Admins | email |

`leftJoysticks`, `rightJoysticks`, `steeringKits` are JSON array strings (e.g. `["803398","803402"]`).
`deleteBrand` soft-deletes (sets active=false). `deleteKit` soft-deletes (sets status='hidden').

## Key Technical Details

### gasApi.ts ‚Äî Critical pattern
```typescript
// MUST pass the handler-attached `runner` object, NOT google.script.run
function gasRun<T>(fn: (run: typeof google.script.run) => void): Promise<T> {
  return new Promise((resolve, reject) => {
    const runner = google.script.run
      .withSuccessHandler((result: unknown) => resolve(result as T))
      .withFailureHandler((err: { message: string }) => reject(new Error(err.message)));
    fn(runner);  // ‚Üê runner, not google.script.run
  });
}
```

### Build pipeline
```bash
./scripts/build.sh     # Vite bundles client ‚Üí gas/index.html (single file, ~241 kB)
npx clasp push         # Push gas/ to GAS (answer "y" to manifest prompt)
npx clasp deploy --deploymentId <ID> --description "VX.X"  # Update live URL
```

## Features Implemented

### Sales UI (role: sales)
1. **BrandGrid** ‚Äî responsive logo grid with search filter
2. **ModelList** ‚Äî model table with kit status badges (Available / Coming Soon)
3. **KitDetail** ‚Äî full kit breakdown:
   - Base Kit (614168) ‚Äî 7 fixed components
   - Boom/Arm Cable (10m = 803411, 15m = 803410)
   - Joysticks (left/right A9 variants with Default badge)
   - Configuration File (machine-specific part number)
   - Machine Cable Kit (6-digit part number + description)
   - Optional Add-ons (feeder valve / extra QIO module)
   - Steering Kits (CEX/MEX/WEX machine-specific)
   - Prerequisites & Limitations
4. **KitRequestModal** ‚Äî request a missing kit with optional note

### Admin UI (role: admin)
1. **Kit List** ‚Äî sortable/filterable table; edit/delete actions
2. **Kit Editor** ‚Äî full form to create/edit kits (all fields)
3. **Brand Manager** ‚Äî two-panel brand + model CRUD
4. **Request List** ‚Äî manage kit requests with status workflow (new ‚Üí acknowledged ‚Üí in_progress ‚Üí resolved)
5. **Preview Mode** ‚Äî toggle to see the app as a sales user (amber banner, "‚Üê Back to Admin" button)

## Common Tasks

### Add a new field to Kit
1. `gas/Types.ts` ‚Äî add to `Kit` interface and `KIT_COLS` map (increment column number)
2. `gas/DB.ts` ‚Äî update `rowToKit()` and `saveKit()` (add to 16-col read/write)
3. `gas/Setup.ts` ‚Äî update Kits header in `setupDatabase()`, add migration function
4. `client/src/types.ts` ‚Äî add field to `Kit` interface
5. `client/src/components/admin/KitEditor.tsx` ‚Äî add form field
6. `client/src/components/sales/KitDetail.tsx` ‚Äî display field
7. Run migration function in GAS editor for existing deployments
