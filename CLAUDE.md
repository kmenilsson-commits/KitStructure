# QTC Kit Configurator — Claude Context

## What This Is
A **Google Apps Script web app** for the Steelwrist sales team to look up whether a Quantum Connect (QTC) installation kit exists for a specific excavator brand and model. The backend is Google Sheets; the frontend is React + Vite bundled into a single HTML file served by GAS HtmlService.

**Live URL:** deployed as a Google Workspace domain-restricted web app (execute as user deploying, access: domain).

## Google Sheets Database

**Spreadsheet ID** stored in GAS Script Properties (`SPREADSHEET_ID`). 5 sheets:

| Sheet | Columns |
|-------|---------|
| Brands | id, name, logoFilename, active, createdAt |
| Models | id, brandId, brandName, name, tonnage, machineType, createdAt |
| Kits | id, modelIds, status, cableLength, leftJoysticks, rightJoysticks, needsFeederValves, needsExtraQio, steeringKits, configPartNumber, prerequisites, limitations, updatedAt, updatedBy, cableKitPartNumber, cableKitDescription, joystickRollerType, joystickButtonType, joystickConnectorType, joystickConnectorPins, safetyGateSignal, machineType |
| KitRequests | id, brandName, modelName, requestedBy, note, status, createdAt, adminNote |
| Admins | email |

`leftJoysticks`, `rightJoysticks`, `steeringKits`, `modelIds` are JSON array strings.
`deleteBrand` soft-deletes (sets active=false). `deleteKit` soft-deletes (sets status='hidden').

### Kit ↔ Model relationship
- A kit can be linked to **multiple models** — `modelIds` is a JSON string array of model UUIDs.
- `normalizeModelIds()` in DB.ts handles old single-UUID rows for backward compatibility.

### Kit columns 17–22 (added V1.8)
Admin-only reference fields about the original machine joystick: `joystickRollerType`, `joystickButtonType`, `joystickConnectorType`, `joystickConnectorPins`, `safetyGateSignal`, `machineType`.
Run `migrateJoystickInfoColumns()` once in the GAS editor on existing deployments.

## Key Technical Details

### gasApi.ts — Critical pattern
```typescript
// MUST pass the handler-attached `runner` object, NOT google.script.run
function gasRun<T>(fn: (run: typeof google.script.run) => void): Promise<T> {
  return new Promise((resolve, reject) => {
    const runner = google.script.run
      .withSuccessHandler((result: unknown) => resolve(result as T))
      .withFailureHandler((err: { message: string }) => reject(new Error(err.message)));
    fn(runner);  // ← runner, not google.script.run
  });
}
```

### Build pipeline
```bash
./scripts/build.sh     # Vite bundles client → gas/index.html (single file, ~615 kB)
npx clasp push         # Push gas/ to GAS (answer "y" to manifest prompt)
npx clasp deploy --deploymentId <ID> --description "VX.X"
```

**Deployment ID:** `AKfycbyvaSwCLUfBZ2UD-JxSScVruY2I_Z63eDBiLt4aldnTzUjZBYf_Vw7zsRptMRB_VzJU`

### GAS TypeScript
- **No import/export** — all files share a global namespace
- Types in `Types.ts` are available in all other GAS files automatically

### vite.config.ts
- `assetsInlineLimit: 1024 * 1024` — all assets ≤1 MB are inlined as base64 (needed for bundled brand logos)
- `viteSingleFile` plugin — bundles all JS + CSS into a single HTML file

## Features Implemented

### Sales UI (role: sales)
1. **BrandGrid** — responsive logo grid with search filter; **"Show only brands with available kits" toggle** filters to brands with at least one available/coming_soon kit
2. **ModelList** — model table with kit status badges (Available / Coming Soon); per-row loading spinner; chevron affordance; "No kit — tap to request" for missing kits
3. **KitDetail** — full kit breakdown: Base Kit (614168, collapsible), Boom/Arm Cable, Joysticks, Config File, Machine Cable Kit, Optional Add-ons, Steering Kits, Prerequisites & Limitations. Request button only shown when kit is null.
4. **KitRequestModal** — request a missing kit with optional note

### Admin UI (role: admin)
1. **Kit List** — sortable/filterable table; edit/delete actions
2. **Kit Editor** — full form; multi-model selector (pick brand first, tick models); joystick info section (6 single-select fields); accepts `prefillModelId` prop to pre-populate from a kit request
3. **Brand Manager** — two-panel brand + model CRUD; clicking edit pen selects that brand; Logo URL field accepts drag-drop of image file (stored as base64 data URI)
4. **Request List** — manage kit requests with status workflow; **"Create Kit" button** on each row opens KitEditor pre-filled with the matching brand/model
5. **Preview Mode** — toggle to see the app as a sales user (amber banner)

## Brand Logos

Logo files live in `Documents/Brand Logos/` (27 brands). Best variant for each brand is copied to `client/src/assets/logos/` and imported as base64 data URIs via `client/src/data/brandLogos.ts`.

Resolution priority:
1. **Explicit URL or base64 data URI** — `isExplicitLogoUrl(brand.logoFilename)` (admin override via Brand Manager; drag-drop stores as `data:` URI)
2. **Bundled PNG** — `BUNDLED_LOGOS[brand.name]` covers: Airman, Atlas, Bobcat, CAT, Case, Develon, Doosan, Hidromek, Hitachi, Hyundai, JCB, John Deere, Kobelco, Komatsu, Kubota, Liebherr, Link-Belt, Mecalac, Sany, Sumitomo, Sunward, Takeuchi, Terex, Volvo, Wacker Neuson, XCMG, Yanmar
3. **Clearbit** — `https://logo.clearbit.com/{domain}` (unreliable free tier)
4. **Google favicon** — `https://www.google.com/s2/favicons?domain={domain}&sz=128`
5. **Coloured initials circle** — last resort

`BrandLogo` is a shared component (exported from `BrandGrid.tsx`, used in `ModelList.tsx` and `BrandManager.tsx`) with three-stage error fallback. `isExplicitLogoUrl()` is exported from `BrandGrid.tsx`.

To add a new logo: copy PNG to `client/src/assets/logos/`, add import + entry in `brandLogos.ts`, rebuild.

## Common Tasks

### Add a new field to Kit
1. `gas/Types.ts` — add to `Kit` interface and `KIT_COLS` map (currently up to col 22)
2. `gas/DB.ts` — update `rowToKit()` and `saveKit()` (update col count in setValues/appendRow)
3. `gas/Setup.ts` — update Kits header in `setupDatabase()`, add migration function
4. `client/src/types.ts` — add field to `Kit` interface
5. `client/src/components/admin/KitEditor.tsx` — add to `buildEmptyKit()` and add form field
6. `client/src/components/sales/KitDetail.tsx` — display field (if sales-facing)
7. Run migration function in GAS editor for existing deployments
